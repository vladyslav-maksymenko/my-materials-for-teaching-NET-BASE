# Керівництво по практичним завданням

## Структура файлів

### PracticalTasks.cs
Містить:
- **Шаблони для завдань 1-4** (в коментарях) - базові класи для студентів
- **Повні вирішення всіх 6 завдань** - для швидкої допомоги студентам

### PracticalTasksDemo.cs
Демонстрація використання всіх завдань з прикладами виводу.

## Як використовувати

### Для студентів (завдання 1-4):
1. Відкрийте `PracticalTasks.cs`
2. Знайдіть шаблон завдання в коментарях (наприклад, `// ЗАВДАННЯ 1`)
3. Розкоментуйте шаблон та реалізуйте необхідні оператори/властивості
4. Використовуйте готове вирішення як довідник

### Для викладача:
1. Швидка допомога студентам:
   - Покажіть готове вирішення з `PracticalTasks.cs`
   - Запустіть `PracticalTasksDemo.cs` для демонстрації

2. Перевірка роботи:
   - Запустіть `dotnet run` - побачите демонстрацію всіх завдань
   - Або використовуйте `PracticalTasksDemo.RunAllDemos()` в своєму коді

## Детальний опис завдань

### Завдання 1: Співробітник (Employee)
**Що потрібно:**
- Перевантажити `+`, `-` (зміна зарплати)
- Перевантажити `==`, `!=`, `<`, `>` (порівняння зарплат)
- Перевизначити `Equals()` та `GetHashCode()`
- Використати властивості

**Ключові моменти:**
- Оператори `+` та `-` повертають новий об'єкт (не змінюють оригінал)
- `==` і `!=` завжди перевантажуються разом
- При перевантаженні `==` обов'язково перевизначити `Equals()`

### Завдання 2: Матриця (MatrixTask)
**Що потрібно:**
- Перевантажити `+`, `-` (додавання/віднімання матриць)
- Перевантажити `*` (множення матриць та на число)
- Перевантажити `==`, `!=`
- Реалізувати індексатор `this[int row, int col]`
- Використати властивості

**Ключові моменти:**
- Перевіряти розміри матриць перед операціями
- Індексатор має валідувати індекси
- Множення матриць: кількість стовпців першої = кількість рядків другої

### Завдання 3: Місто (City)
**Що потрібно:**
- Перевантажити `+`, `-` (зміна населення)
- Перевантажити `==`, `!=`, `<`, `>` (порівняння за населенням)
- Перевизначити `Equals()` та `GetHashCode()`
- Використати властивості

**Ключові моменти:**
- Аналогічно до завдання 1, але для населення
- Перевіряти, щоб населення не було від'ємним

### Завдання 4: Кредитна картка (CreditCard)
**Що потрібно:**
- Перевантажити `+`, `-` (зміна балансу)
- Перевантажити `==`, `!=` (порівняння за CVC)
- Перевантажити `<`, `>` (порівняння за балансом)
- Перевизначити `Equals()` та `GetHashCode()`
- Використати властивості

**Ключові моменти:**
- `==` порівнює за CVC кодом
- `<` і `>` порівнюють за балансом
- Різні критерії порівняння в одному класі

### Завдання 5: Currency (Конвертація валют)
**Повне вирішення включено**

**Що демонструє:**
- Неявне перетворення: `Currency c = 50.0;`
- Явне перетворення: `double d = (double)c;`
- Оператори `+`, `-` для різних валют
- Властивості для курсу обміну

**Ключові моменти:**
- `implicit` - неявне перетворення (без вказання типу)
- `explicit` - явне перетворення (з вказанням типу)
- Конвертація через USD як базову валюту

### Завдання 6: Player (Гравець)
**Повне вирішення включено**

**Що демонструє:**
- Автоматичні властивості
- Логіка в `set` аксесорі
- Автоматичне підвищення рівня при досягненні порогу

**Ключові моменти:**
- `set` аксесор викликає метод `UpdateLevel()`
- Рівень обчислюється автоматично: `Level = (Score / 100) + 1`
- Приклад використання логіки в властивостях

## Типові помилки студентів

1. **Забувають перевантажити `!=` разом з `==`**
   - Рішення: завжди перевантажувати пари операторів

2. **Не перевизначають `Equals()` при перевантаженні `==`**
   - Рішення: обов'язково перевизначити `Equals()` та `GetHashCode()`

3. **Змінюють оригінальний об'єкт в операторах `+`, `-`**
   - Рішення: створювати новий об'єкт, не змінювати оригінальний

4. **Не валідують індекси в індексаторах**
   - Рішення: завжди перевіряти межі масиву

5. **Не перевіряють розміри матриць перед операціями**
   - Рішення: додати перевірки в оператори

## Швидкі підказки для допомоги студентам

### Як перевантажити оператор:
```csharp
public static ReturnType operator operator_symbol(Type1 op1, Type2 op2)
{
    // логіка
    return result;
}
```

### Як створити індексатор:
```csharp
public ReturnType this[IndexType index]
{
    get { /* логіка */ }
    set { /* логіка */ }
}
```

### Як створити властивість:
```csharp
private Type _field;
public Type Property
{
    get => _field;
    set => _field = value;
}
```

### Як перевизначити Equals:
```csharp
public override bool Equals(object? obj)
{
    if (obj is MyType other)
        return this == other;
    return false;
}
```
